Task 11 – Testing and debugging the Community Event Portal with Chrome DevTools

How to Use Chrome DevTools

1

Open DevTools

Press F12 or right-click any element and choose Inspect. You can also use Ctrl+Shift+I (Windows) or Cmd+Opt+I (Mac).

2

Inspect an Element

Click the cursor icon (top-left of DevTools) then click any element on the page. The Elements panel shows its HTML and the Styles pane shows all CSS rules applied to it.

3

Live-Edit CSS

In the Styles pane, click any property value and type a new value. Changes are instant. Try changing background-color or font-size on the header to see it update live.

4

Box Model Inspector

In the Computed tab, scroll to find the box model diagram. It shows the exact pixel values for margin, border, padding, and content.

5

Device Toolbar (Responsive)

Click the phone/tablet icon (Ctrl+Shift+M) to open the responsive design toolbar. Select a device preset (e.g. iPhone 14) or drag the edges to any custom width.

6

Network Tab

Open the Network tab and reload the page (F5). All loaded resources appear here. Check that task11_external.css shows status 200. A 404 means the file was not found.

Bug Hunt: Fix This Broken Card (DevTools Practice)

⚠️ This card is too wide! (Intentional Bug)

This card has width: 1200px applied, which breaks the layout on smaller screens. Open DevTools → Inspect this element → find the width: 1200px rule in the Styles pane → uncheck or change it to width: 100% to fix the layout.

How to fix: In DevTools, select the .broken-card element. In the Styles panel, find width: 1200px and change it to width: 100%. The card will snap back into the page width immediately. Remember: changes made in DevTools are not saved permanently — apply the fix in your CSS file to make it stick.
