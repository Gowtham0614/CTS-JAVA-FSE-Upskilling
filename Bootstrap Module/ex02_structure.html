Exercise 2: Bootstrap Structure and Files

Exercise 2.1 – Directory Structure

When you download Bootstrap, the directory looks like:

bootstrap/
├── css/
│   ├── bootstrap.min.css         ← Main stylesheet (minified)
│   ├── bootstrap.css             ← Main stylesheet (readable)
│   ├── bootstrap-grid.min.css    ← Grid only
│   ├── bootstrap-reboot.min.css  ← CSS reset only
│   └── bootstrap-utilities.min.css ← Utility classes only
│
├── js/
│   ├── bootstrap.bundle.min.js   ← JS + Popper.js (recommended)
│   ├── bootstrap.min.js          ← JS only (requires separate Popper)
│   └── bootstrap.esm.min.js      ← ES Module version
│
└── (Bootstrap Icons is a separate package)
    └── bootstrap-icons/
        ├── font/
        │   └── bootstrap-icons.css
        └── icons/ (SVG files)
📁 css/
Contains all compiled CSS files. Use bootstrap.min.css in production. Separate files like bootstrap-grid.css let you load only what you need.
📁 js/
Contains JavaScript plugins. bootstrap.bundle.min.js includes Popper.js for dropdowns/tooltips. Use this unless you manage Popper separately.
📦 Bootstrap Icons
A separate icon library with 1800+ SVG icons. Install via npm or CDN. Includes font files and individual SVGs.

Exercise 2.2 – Including bootstrap.bundle.min.js

Place the JS bundle just before </body>:

<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>
